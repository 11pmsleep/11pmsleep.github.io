Webshell

Webshell简介

WebShell就是以ASP、PHP、JSP或者CGI等网页文件形式存在的一种命令执行环境，也可以将其称之为一种网页后门。攻击者在入侵了一个网站后，通常会将这些asp或php后门文件与网站服务器web目录下正常的网页文件混在一起，然后使用浏览器来访问这些后门，得到一个命令执行环境，以达到控制网站服务器的目的（可以上传下载或者修改文件，操作数据库，执行任意命令等）。

要点解释：

  • php的代码要写在 里面，服务器才能认出来这是php代码，然后才去解析php的内容。

  • @ 符号的意思是不报错，因为cmd这个变量没有定义而被直接使用，服务器会提醒cmd变量未定义。

  • $_POST['cmd'];的解释：

    php里面有几个超全局变量， $_GET、$_POST 就是其中之一，意思是用 post的方法接收变量cmd传递来的字符。

    比如$_POST['cmd']接受到cmd=echo 123，则代码就相当于：<?php @eval(echo 123)?>

  • php危险函数：

    • eval()：把字符串作为PHP代码执行
    • exec()： 执行一个外部程序
    • system()：执行外部程序，并且显示输出

  • 如上所述：eval()函数可以将一个字符串当作php的代码执行~~~，于是

    这串代码<?php @eval(echo 123)?>等价于====><?php echo 123?>

    相当于就是通过php执行了一条php的代码命令：echo 123

冰蝎-Behinder

冰蝎是一款基于 Java 开发的动态加密通信流量的新型 Webshell 客户端。老牌 Webshell 管理神器——中国菜刀的攻击流量特征明显，容易被各类安全设备检测，实际场景中越来越少使用，加密Webshell 正变得日趋流行。由于通信流量被加密，传统的 WAF、IDS 设备难以检测，给威胁狩猎带来较大挑战。

冰蝎其最大特点就是对交互流量进行对称加密，且加密秘钥是由随机数函数动态生成，因此该客户端的流量几乎无法检测。冰蝎目前最新版本为 v3，兼容性已经日益完善，包括虚拟终端、Socks 代理、文件管理、反弹 shell、数据库管理等强大的功能，方便部署使用。

  • 有流量动态加密

  • 冰蝎对上传的webshell有要求 具体要求要自己去了解

  • 要上传冰蝎的shell，才可以连上冰蝎

注意新版冰蝎4.0的用法：[kali安装jdk8和切换jdk版本和4.0版本冰蝎的使用_Mauro_K的博客-CSDN博客](https://blog.csdn.net/qq_44029310/article/details/126086895#:~:text=二、4.0版本的冰蝎使用,1）下载之后，解压进入目录。 2）打开命令行，使用以下命令开启。)

哥斯拉-Godzilla

不过，此哥斯拉非彼哥斯拉，他是继菜刀、蚁剑、冰蝎之后具有更多优点的 Webshell 管理工具，java语言开发，如名称一样，他的“凶猛”之处主要体现在：

  • 全部类型的 shell 能绕过市面大部分的静态查杀；
  • 流量加密能绕过过市面绝大部分的流量 Waf；
  • Godzilla 自带的插件是冰蝎、蚁剑不能比拟的；

  • 哥斯拉有生成脚本的功能，要使用生成的脚本连接

总结流量特征：

  • 蚁剑对流量有base64加密，但是有些关键代码是没有被加密的，比如：PHP 中的 ini_set；ASP 中的 OnError,response。
  • 流量最中明显的特征为@ini_set("display_errors", "0");这段代码基本是所有WebShell客户端链接PHP类WebShell都有的一种代码。使用不同的编码方式，payload 的特征都会有所不同，但殊途同归，几乎都存在 @ini_set("display_errors", "0"); 的不同编码形式。

  • 由于蚁剑中包含了很多加密、绕过插件，所以导致很多流量被加密后无法识别，但是蚁剑混淆加密后还有一个比较明显的特征，即为参数名大多以“_0x.....=”这种形式（下划线可替换为其他）所以，以_0x开头的参数名，后面为加密数据的数据包也可识别为蚁剑的流量特征。

  • 会带有base64编码解码等字符特征

WAF 的绕过方法总结

  1. 大小写绕过；这个说的很多了，有的 WAF 就是只过滤小写的关键字，例如 script,union,select等；
  2. 双写关键字绕过；
  3. 同价词替换：

and 和 or 有可能不能使用，或者可以试下&&和||能不能用；

=不能使用的情况，可以考虑尝试<、>，因为如果不小于又不大于，那就是等于了；
空格不能使用（常见），可以使用如下符号进行替换：%20 %09 %0a %0b %0c %0d %a0 /**/；

  1. 编码绕过，16 进制绕过，URL 编码，Unicode 编码绕过等；

  2. 使用注释符绕过（都可以去尝试）

常见的用于注释的符号有哪些：//, -- , /**/, #, --+,-- -, ;，--a
内联注释用的更多，它有一个特性/!**/只有 MySQL 能识别(/*！*/表示注释里面的语句会被执行)。
